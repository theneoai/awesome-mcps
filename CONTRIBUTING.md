# 贡献指南

感谢你有兴趣为 Awesome MCPs 贡献内容！

## 如何提交

1. Fork 本仓库
2. 在对应分类的 `categories/<category>/README.md` 中添加条目
3. 同步更新根目录 `README.md` 中的对应分类表格
4. 提交 Pull Request，标题格式：`Add: <MCP名称>`

## 入选标准

提交的 MCP 需满足以下条件：

- **开源**：代码公开，许可证明确（MIT、Apache 2.0、BSD 等）
- **文档完整**：有清晰的 README、安装说明和使用示例
- **维护活跃**：过去 6 个月内有代码提交或对 Issue 有响应
- **功能实用**：解决真实需求，非玩具项目
- **质量可靠**：无明显安全问题，代码质量良好

## 分类说明

| 目录 | 适用内容 |
|------|---------|
| `categories/development/` | 代码托管、CI/CD、容器、IDE 集成等开发工具 |
| `categories/productivity/` | 项目管理、日历、笔记、任务管理等效率工具 |
| `categories/data/` | 数据库、数据仓库、向量存储、数据分析 |
| `categories/communication/` | 消息、邮件、通知、会议等通信平台 |
| `categories/ai/` | AI 模型、推理平台、ML 框架集成 |
| `categories/security/` | 密钥管理、漏洞扫描、安全审计 |
| `categories/utilities/` | 搜索、爬虫、文件处理、通用工具 |

## 条目格式

在分类 README 的表格中按如下格式添加：

```markdown
| [名称](链接) | 一句话描述（中文） | 实现语言 | ![stars](badge) |
```

- 描述保持简洁，不超过 30 字
- 按首字母或知名度排序
- 官方维护的 MCP 请在描述后加 `（官方）`

## 行为准则

请保持友善、包容的交流氛围。不接受任何形式的骚扰或歧视性言论。
